06

Page definition

Creating the rules-based UX for defining and managing page structures in web analytics.

Rules UX Web UXCam
Cover image placeholder

Overview

Web analytics depends on accurate page identification. Unlike mobile apps with discrete screens, web pages are defined by URLs that can be complex, dynamic, and inconsistent. This project designed the system that lets users define and manage how pages are identified and grouped within UXCam's web product.

Problem

Page definition for web analytics involves inherent complexity:

  • URLs contain query parameters, fragments, and dynamic segments that need flexible matching
  • Single-page applications blur the line between pages and views
  • Users needed to group similar URLs without writing regex
  • The system had to work automatically while allowing manual override

Process

I mapped the different URL patterns our customers encounter and designed a rules engine that could handle them. The design process involved close collaboration with engineering to understand pattern matching capabilities and with customers to validate that the mental model made sense.

We iterated on the rule builder interface through multiple rounds of usability testing, focusing on making regex-level power accessible without requiring regex knowledge.

Solution

The page definition system uses a visual rule builder with conditions like "URL contains," "URL starts with," and wildcard matching. Users can preview which URLs match their rules in real-time, create page groups for aggregate analysis, and set priorities when rules overlap. Auto-detection handles common cases with manual override available.

Outcome

The page definition feature enabled accurate web analytics for customers with complex site architectures. The visual rule builder made the feature accessible to non-technical users while providing enough flexibility for advanced use cases.
