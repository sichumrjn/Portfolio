Coded prototype of signup experience

Building a high-fidelity HTML/CSS prototype to test and validate the new signup flow before engineering handoff.

Prototyping HTML/CSS UXCam
Cover image placeholder

Overview

After designing the new signup-to-activation flow, we needed a way to test the experience with real interactions — clicks, transitions, form validation — that static mockups couldn't capture. I built a coded prototype in HTML/CSS/JS that replicated the full signup journey.

Problem

Design tools can only approximate the real experience. For a flow as critical as signup, we needed higher fidelity testing:

  • Static prototypes missed timing and transition nuances
  • Stakeholders had difficulty imagining the final experience from mockups alone
  • We needed to test with real form interactions and validation feedback
  • Engineering needed a concrete reference beyond design specs

Process

I translated the Figma designs into clean, semantic HTML with CSS that closely matched our design system. The prototype included form validation, step transitions, loading states, and responsive behavior.

This was shared with users for remote testing sessions and served as a living spec that engineers could inspect for implementation details.

Solution

The coded prototype covered the complete signup flow: account creation, workspace setup, SDK integration guide, and the first-run experience. Every state — empty, loading, error, success — was implemented to give users and stakeholders a complete picture.

Outcome

Testing with the coded prototype revealed interaction issues we'd missed in static mockups. It accelerated engineering handoff by reducing ambiguity and became a reusable reference for future onboarding iterations.
