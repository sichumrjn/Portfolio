05

Heatmap creation for web

Designing the heatmap generation workflow for UXCam's web analytics platform.

Data Viz Web UXCam
Cover image placeholder

Overview

As UXCam expanded from mobile to web analytics, heatmaps were a core feature that needed to be designed for a fundamentally different context. Web pages are dynamic, responsive, and structurally more complex than mobile screens. This project defined how users create, configure, and interpret heatmaps for web.

Problem

Translating heatmaps from mobile to web introduced new challenges:

  • Web pages have variable layouts across breakpoints and viewport sizes
  • Dynamic content (SPAs, lazy loading) complicates page capture
  • Users needed to specify which page variant to analyze
  • The creation flow needed to handle both simple and complex use cases

Process

I researched how other analytics tools handled web heatmaps, identified gaps, and mapped UXCam's unique advantages. Working with engineering, I defined what was technically feasible for the initial release and designed a progressive disclosure model that could grow over time.

Prototypes were tested with existing UXCam customers to validate the creation workflow and data visualization approach.

Solution

The heatmap creation flow guides users through page selection, viewport configuration, and data range settings. The visualization supports click, scroll, and attention heatmaps with toggleable overlays. A responsive preview lets users switch between device sizes to see how interaction patterns differ.

Outcome

The web heatmap feature launched as a key differentiator for UXCam's web product. The creation workflow tested well with users, and the progressive complexity model allowed power users to access advanced configurations without overwhelming new users.
